# 等距回合制战棋 Demo 策划案

## 0. 项目定位

项目暂名：绵羊地下城：狂战士试炼

项目类型：等距视角回合制战术 RPG Demo

参考目标：学习 Wakfu 的战斗结构，包括等距网格、AP/MP 行动资源、技能射程、范围预览、站位、朝向、背击、状态效果、职业机制和副本战斗节奏。

开发目标：做一个 10-15 分钟可通关的小型战斗 Demo。玩家控制 1 名狂战士，通过 3 个战斗房间，最后击败 Boss。

学习边界：

- 本地学习阶段可以临摹 Wakfu 的角色比例、动作节奏、技能结构和 UI 布局。
- 不建议直接公开使用 Wakfu 的角色、职业名、技能名、图标、地图、音效、音乐、截图或可识别设定。
- 如果以后要发布、开源、参赛、放作品集，需要把角色、技能、美术、UI、怪物和世界观全部替换成原创版本。

## 1. Demo 最终体验

玩家进入一个绵羊主题地下城，控制一个近战狂战士角色。每回合玩家有固定 AP 和 MP，需要选择移动位置、技能顺序和攻击方向。狂战士越受伤越危险，也越强。玩家需要利用冲锋、流血、背击、击退和怒气机制，在 3 场战斗中击败敌人。

玩家应该能感受到：

- 每回合行动点有限，不能无脑放技能。
- 移动和站位会影响伤害和生存。
- 血量越低，狂战士越强，但也更容易死亡。
- 敌人有不同机制，不是只会走过来攻击。
- Boss 有明显预警技能，玩家需要躲避或利用地形反制。

## 3. 推荐制作工具

推荐使用：

- 引擎：Godot 4
- 语言：GDScript
- 地图：Godot TileMapLayer
- 美术：Aseprite、Krita 或 Photoshop
- 数据：Godot Resource 或 JSON
- 平台：PC

原因：

- Godot 做 2D 等距战棋足够轻量。
- GDScript 上手快。
- TileMapLayer 适合做格子地图。
- Demo 阶段不需要 Unity 或 Unreal 的大型工程复杂度。

## 4. 核心战斗规则

### 4.1 地图与格子

地图是等距菱形网格。底层逻辑不要用屏幕坐标计算，而是用格子坐标。

示例：

```text
格子坐标：(x, y)
地图尺寸：10 x 10
单位占用：1 个格子
障碍物：不可通行，不可站立
敌我单位：不可重叠
```

每个格子需要记录：

```text
是否可通行
是否有单位
是否是障碍物
是否是特殊格
是否处于技能预警范围
```

### 4.2 回合流程

每个单位按速度或固定顺序行动。第一版可以用固定顺序，降低难度。

推荐第一版顺序：

```text
玩家回合
敌人 1 回合
敌人 2 回合
敌人 3 回合
回到玩家回合
```

单个单位回合流程：

```text
回合开始
恢复 AP / MP
处理状态效果，比如流血掉血
玩家选择移动和技能
消耗 AP / MP
执行技能效果
点击结束回合
进入下一个单位
```

### 4.3 AP / MP / 怒气

AP：行动点，用于释放技能。

MP：移动点，用于移动。

怒气：狂战士职业资源，用于强化技能。

狂战士初始数值：

| 属性 | 数值 |
|---|---:|
| HP | 100 |
| AP | 6 |
| MP | 3 |
| 怒气 | 0 |
| 最大怒气 | 100 |

怒气获得规则：

| 行为 | 怒气 |
|---|---:|
| 受到伤害 | +10 |
| 造成近战伤害 | +5 |
| 击杀敌人 | +20 |
| 生命低于 50% 时每回合开始 | +10 |

怒气用途：

- 部分技能消耗怒气。
- 怒气越高，部分技能伤害越高。
- 怒气满时进入 1 回合狂暴状态。

### 4.4 移动规则

每移动 1 格消耗 1 MP。

不能移动到：

- 障碍物格
- 敌人占用格
- 友方占用格
- 地图外

如果单位与敌人相邻，离开敌人身边时触发锁定判定。第一版可以先做简单规则：

```text
如果从敌人相邻格离开，额外消耗 1 MP。
如果 MP 不足，则不能离开。
```

这样可以模拟 Wakfu 里近战贴身的压迫感，但实现更简单。

### 4.5 朝向与背击

单位有 4 个朝向：

```text
上
下
左
右
```

攻击方向影响伤害：

| 攻击位置 | 伤害倍率 |
|---|---:|
| 正面 | 100% |
| 侧面 | 110% |
| 背面 | 125% |

第一版只需要在攻击结算时判断攻击者相对目标的位置。

### 4.6 状态效果

第一版只做 5 个状态：

| 状态 | 效果 |
|---|---|
| 流血 | 每回合开始受到固定伤害 |
| 狂暴 | 伤害提高，受到伤害也提高 |
| 护甲 | 抵消一定伤害 |
| 眩晕 | 跳过 1 回合 |
| 蓄力 | 下一回合释放强力技能 |

## 5. 狂战士职业设计

### 5.1 职业定位

狂战士是近战高风险输出职业。特点是：

- 贴身战斗强。
- 生命越低越危险。
- 可以用冲锋快速接近敌人。
- 可以用流血和范围斩击处理多个敌人。
- 需要在卖血和保命之间做选择。

### 5.2 职业被动

技能名：血怒

效果：

```text
每损失 10% 最大生命，造成伤害提高 3%。
生命低于 50% 时，回合开始额外获得 10 怒气。
生命低于 25% 时，近战攻击造成伤害的 20% 转化为治疗。
```

设计目的：

让玩家主动思考是否承受伤害换取输出，而不是满血无脑打。

### 5.3 技能表

#### 技能 1：斩击

| 项目 | 数值 |
|---|---|
| 类型 | 近战单体 |
| 消耗 | 3 AP |
| 射程 | 1 |
| 范围 | 单体 |
| 冷却 | 无 |
| 基础伤害 | 18 |
| 附加效果 | 命中后获得 5 怒气 |

用途：

基础输出技能。玩家每回合大概率会使用。

#### 技能 2：冲锋

| 项目 | 数值 |
|---|---|
| 类型 | 位移攻击 |
| 消耗 | 2 AP + 1 MP |
| 射程 | 直线 3 格 |
| 范围 | 单体 |
| 冷却 | 2 回合 |
| 基础伤害 | 14 |
| 附加效果 | 移动到目标相邻格，并造成伤害 |

限制：

- 只能直线冲锋。
- 路径中不能有障碍物。
- 目标必须是敌人。

用途：

快速接近远程敌人，或者调整站位打背击。

#### 技能 3：撕裂

| 项目 | 数值 |
|---|---|
| 类型 | 近战单体 |
| 消耗 | 3 AP |
| 射程 | 1 |
| 范围 | 单体 |
| 冷却 | 无 |
| 基础伤害 | 10 |
| 附加效果 | 施加 2 回合流血，每回合 8 伤害 |

用途：

对高血量敌人和 Boss 使用。让玩家理解持续伤害。

#### 技能 4：血怒斩

| 项目 | 数值 |
|---|---|
| 类型 | 高伤害单体 |
| 消耗 | 4 AP |
| 射程 | 1 |
| 范围 | 单体 |
| 冷却 | 1 回合 |
| 基础伤害 | 22 |
| 附加效果 | 每有 20 怒气，额外 +10% 伤害 |

用途：

狂战士核心爆发技能。怒气高时使用收益最大。

#### 技能 5：旋风斩

| 项目 | 数值 |
|---|---|
| 类型 | 近战范围 |
| 消耗 | 5 AP |
| 射程 | 自身 |
| 范围 | 周围 8 格 |
| 冷却 | 2 回合 |
| 基础伤害 | 16 |
| 附加效果 | 如果命中 2 个以上敌人，获得 15 怒气 |

用途：

处理被包围的情况，让玩家主动站到多个敌人中间换取收益。

### 5.4 第一版推荐连招

```text
冲锋接近敌人 -> 斩击
撕裂挂流血 -> 下回合血怒斩
被多个敌人包围 -> 旋风斩
低血量高怒气 -> 血怒斩爆发
```

## 6. 敌人设计

### 6.1 绵羊幼崽

定位：教学近战怪。

| 属性 | 数值 |
|---|---:|
| HP | 35 |
| AP | 4 |
| MP | 3 |
| 攻击 | 8 |

行为：

```text
如果能攻击玩家，使用普通攻击。
如果不能攻击，向玩家移动。
```

### 6.2 卷毛羊

定位：高血量阻挡怪。

| 属性 | 数值 |
|---|---:|
| HP | 60 |
| AP | 4 |
| MP | 2 |
| 攻击 | 6 |
| 护甲 | 8 |

技能：卷毛护甲

```text
每 2 回合给自己 10 护甲。
```

用途：

教玩家不要只打正面，要绕背或用流血。

### 6.3 冲撞羊

定位：直线冲锋怪。

| 属性 | 数值 |
|---|---:|
| HP | 45 |
| AP | 5 |
| MP | 3 |
| 攻击 | 10 |

技能：羊角冲撞

```text
如果玩家在同一直线 4 格内，冲向玩家并造成 12 伤害，击退 1 格。
否则向玩家移动。
```

用途：

教玩家不要站在直线上，学习观察敌人攻击范围。

### 6.4 牧羊术士

定位：远程辅助怪。

| 属性 | 数值 |
|---|---:|
| HP | 30 |
| AP | 5 |
| MP | 2 |
| 攻击 | 7 |

技能：

```text
羊毛飞弹：射程 4，造成 7 伤害。
羊毛护盾：给一个友方 10 护甲，冷却 2 回合。
```

用途：

迫使玩家用冲锋处理远程单位。

## 7. Boss 设计

Boss 名称：地下绵羊王

定位：召唤 + 蓄力冲锋 + 范围震地。

| 属性 | 数值 |
|---|---:|
| HP | 180 |
| AP | 6 |
| MP | 2 |
| 普通攻击 | 12 |

### Boss 技能

#### 技能 1：王角重击

```text
射程 1
造成 14 伤害
如果从正面命中，额外击退 1 格
```

#### 技能 2：召唤羊群

```text
每 3 回合召唤 1 只绵羊幼崽
最多同时存在 2 只召唤羊
```

#### 技能 3：蓄力冲锋

回合 A：

```text
Boss 进入蓄力状态。
地图上显示一条直线红色预警区域。
```

回合 B：

```text
Boss 沿预警直线冲锋。
命中玩家造成 28 伤害并击退 2 格。
如果 Boss 撞到石柱，Boss 眩晕 1 回合。
```

用途：

这是 Boss 战核心机制。玩家需要躲开红色区域，或者把 Boss 引导去撞石柱。

#### 技能 4：羊王震地

```text
Boss 生命低于 50% 后解锁。
以自身为中心，周围 8 格造成 16 伤害。
冷却 3 回合。
```

## 8. 副本设计

### 8.1 房间 1：入口羊圈

目的：

教玩家移动、普攻、背击。

地图：

```text
尺寸：8 x 8
障碍物：2 个木栅栏
敌人：2 只绵羊幼崽
```

胜利条件：

```text
击败所有敌人。
```

玩家应该学会：

- 点击格子移动。
- 使用斩击攻击。
- 绕到敌人背后伤害更高。

### 8.2 房间 2：卷毛通道

目的：

教玩家处理护甲、冲锋、远程怪。

地图：

```text
尺寸：10 x 10
障碍物：石块、羊毛堆
敌人：1 只卷毛羊，1 只冲撞羊，1 只牧羊术士
```

胜利条件：

```text
击败所有敌人。
```

玩家应该学会：

- 不要站在冲撞羊直线上。
- 用冲锋接近牧羊术士。
- 用撕裂处理高护甲敌人。

### 8.3 房间 3：地下羊王祭坛

目的：

综合考验玩家。

地图：

```text
尺寸：12 x 12
障碍物：4 根石柱
敌人：地下绵羊王
召唤物：绵羊幼崽
```

胜利条件：

```text
击败地下绵羊王。
```

失败条件：

```text
玩家死亡。
```

玩家应该学会：

- 观察 Boss 蓄力预警。
- 利用石柱让 Boss 撞晕。
- 在低血量时利用狂战士爆发结束战斗。

## 9. UI 与交互

### 9.1 必须有的 UI

| UI | 作用 |
|---|---|
| HP 条 | 显示当前生命 |
| AP 数字 | 显示剩余行动点 |
| MP 数字 | 显示剩余移动点 |
| 怒气条 | 显示职业资源 |
| 技能栏 | 显示 5 个技能 |
| 结束回合按钮 | 结束当前单位回合 |
| 当前单位标识 | 告诉玩家现在谁在行动 |
| 状态图标 | 显示流血、护甲、眩晕等 |
| 伤害数字 | 攻击后显示伤害 |
| 范围高亮 | 显示可移动和可攻击格子 |

### 9.2 操作方式

```text
左键点击单位：选择单位
左键点击格子：移动
左键点击技能：选择技能
左键点击目标：释放技能
右键：取消当前操作
空格：结束回合
鼠标悬停技能：显示说明
鼠标悬停敌人：显示敌人技能和攻击范围
```

### 9.3 颜色约定

```text
蓝色格子：可移动范围
红色格子：攻击范围
橙色格子：敌人预警范围
绿色格子：可选目标
灰色格子：不可通行
```

## 10. 美术风格

学习阶段目标：

```text
临摹 Wakfu 的等距 2D 表现、角色比例、动画节奏、强轮廓、明亮色块和 UI 信息布局。
```

原创阶段目标：

```text
转为原创法式卡通奇幻风格，避免使用 Wakfu 的可识别角色、职业、图标、UI 和专有设定。
```

### 10.1 角色风格

```text
头身比：2.5 到 3.5 头身
轮廓：外轮廓清晰，剪影夸张
动作：攻击前摇明显，打击动作夸张
表情：夸张、幽默、略带荒诞
颜色：高饱和但不要刺眼
```

### 10.2 狂战士视觉方向

原创版建议：

```text
名称：血角狂战士
武器：巨大斧剑或重剑
服装：破损披风、绷带、铁环、皮革护具
装饰：羊角头盔、红色战纹
主色：赤红、铁灰、暖棕
视觉关键词：粗壮、鲁莽、越战越兴奋
```

### 10.3 绵羊地下城视觉方向

```text
主题：地下牧场 + 古代羊神祭坛
地砖：旧石砖、泥土裂缝、羊毛地毯

灯光：暖黄色火把 + 地下冷色阴影
氛围：奇幻、幽默、轻微荒诞
```

## 11. 音效与反馈

第一版至少需要：

```text
点击 UI 音效
移动脚步音效
普通攻击音效
冲锋音效
受击音效
流血触发音效
Boss 蓄力提示音
胜利音效
失败音效
```

反馈原则：

- 玩家点技能后，范围必须立刻高亮。
- 技能命中后，伤害数字必须清楚。
- 背击、暴击、流血、眩晕要有特殊提示。
- Boss 蓄力必须有明显红色预警和音效。

## 12. 技术模块拆分

建议代码模块：

```text
GridManager：管理格子、坐标转换、障碍物、占位
TurnManager：管理回合顺序
Unit：保存单位属性、状态、朝向
Skill：保存技能消耗、范围、效果
CombatResolver：计算伤害、背击、护甲、状态
Pathfinding：计算可移动范围和路径
AIController：敌人行动逻辑
BattleUI：技能栏、血条、AP/MP、提示
BattleManager：胜负判断、房间切换
```

重要原则：

```text
战斗逻辑用格子坐标计算。
画面表现只负责显示。
不要用角色在屏幕上的像素位置判断战斗结果。
```

## 13. 给小白的开发顺序

不要一开始做完整游戏。按下面顺序做，每一步完成后都能测试。

### 第 1 步：做一个空地图

目标：

```text
屏幕上显示 10x10 等距格子。
鼠标移动到格子上时，格子能高亮。
点击格子能打印格子坐标。
```

完成标准：

```text
你能准确知道鼠标点中了哪个格子。
```

### 第 2 步：放一个玩家单位

目标：

```text
地图上有一个狂战士占位图。
玩家单位站在一个格子里。
点击玩家能选中。
```

完成标准：

```text
选中玩家后显示蓝色可移动范围。
```

### 第 3 步：实现移动

目标：

```text
玩家有 3 MP。
点击 3 格以内的可移动格，角色移动过去。
移动后扣除 MP。
```

完成标准：

```text
不能走出地图。
不能走到障碍物。
不能超过 MP 范围。
```

### 第 4 步：实现 AP 和普攻

目标：

```text
玩家有 6 AP。
斩击消耗 3 AP。
只能攻击相邻敌人。
攻击后敌人扣血。
```

完成标准：

```text
玩家一回合最多可以使用两次斩击。
AP 不足时不能释放技能。
```

### 第 5 步：实现回合结束

目标：

```text
有结束回合按钮。
结束后恢复玩家 AP/MP。
敌人暂时可以不动。
```

完成标准：

```text
每回合 AP 回到 6，MP 回到 3。
```

### 第 6 步：实现敌人 AI

目标：

```text
敌人如果能攻击，就攻击。
敌人如果不能攻击，就向玩家移动。
```

完成标准：

```text
敌人能自动靠近玩家并攻击。
```

### 第 7 步：加入 5 个技能

顺序建议：

```text
斩击
撕裂
冲锋
血怒斩
旋风斩
```

完成标准：

```text
每个技能都有 AP/MP 消耗、范围预览和实际效果。
```

### 第 8 步：加入怒气和状态

目标：

```text
受到伤害获得怒气。
攻击获得怒气。
撕裂能施加流血。
血怒斩能根据怒气提高伤害。
```

完成标准：

```text
玩家能通过卖血和攻击积累怒气，然后打出更高伤害。
```

### 第 9 步：做 3 个房间

目标：

```text
房间 1：2 只绵羊幼崽
房间 2：卷毛羊 + 冲撞羊 + 牧羊术士
房间 3：Boss
```

完成标准：

```text
打完一个房间后进入下一个房间。
```

### 第 10 步：打磨体验

目标：

```text
加动画
加音效
加伤害数字
加技能说明
加敌人预警
调整数值
```

完成标准：

```text
一个第一次玩的玩家能在 10-15 分钟内通关，并能理解基本规则。
```

## 15. 初版数值目标

普通战斗：

```text
房间 1：2-3 回合
房间 2：4-5 回合
Boss 战：6-8 回合
总时长：10-15 分钟
```

难度目标：

```text
玩家允许犯 1-2 次明显错误。
如果玩家完全不管站位，应该会死亡。
如果玩家合理使用冲锋、背击、流血和怒气，应该能稳定通关。
```

## 16. 最终验收标准

这个 Demo 做完后，应该满足：

```text
玩家能完整打通 3 个房间。
玩家能理解 AP/MP 是核心资源。
玩家会为了背击和躲技能调整站位。
狂战士有低血量高收益的职业特色。
至少 4 种敌人有明显差异。
Boss 有可读的蓄力预警和反制方式。
战斗总时长控制在 10-15 分钟。
所有核心操作都有 UI 反馈。
```

如果以上都达成，这个 Demo 就已经完成了学习目标。
